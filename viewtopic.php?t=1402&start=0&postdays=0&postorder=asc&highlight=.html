Versuch's erstmal mit einem http oder ssh(sftp) Server.
Im gegensatz zu diesen Protokollen birgt FTP weitere Schwierigkeiten:
Bei FTP gibt es zwei verbindungen: Eine Kontrollverbindung, die vom Client zum server aufgemacht wird. Deren Port ist fest und kann über NAT/Portforwarding weitergeleitet werden.

Daneben gibt es eineDatenverbindung über einen Port, der von vornherein nicht bekannt ist.
Bei active FTP wird diese Verbindung vom Server initiert, d.h. wenn der Client hinter einer Firewall sitzt, dann funktioniert dies im allgemeinen nicht.
Bei passive FTP wird diese Verbindung vom Client initiiert. Das funktioniert dann auch, wenn der Client hinter einer Firewall sitzt. Wenn nun aber der Server hinter der Firewall sitzt, weißt Du nicht welcher Port mittels Portforwarding weitergeleitet werden muß. Entweder leitest Du nun alle Ports per Portforwarding weiter, was eine Firewall ad absurdum führen würde, oder aber Du verwendest ein alternatives Protokoll.

Zum Kopieren von Daten würde ich dir am besten sftp empfehlen.
