Zu grub2 kann ich Dir nix sagen, aber zu dem irritierenden auto / noauto

In der Datei fstab sind die verschiedenen Parameter durch Leerzeichen getrennt

[Device] [Mount Point] [Filesystem] [Options] [dump] [fsck order]

In Deinem Fall

/dev/fd0 /media/floppy0 auto rw,user,noauto,exec,utf8 0 0

das auto bezieht sich auf das Dateisystem (automatisch erkennen), noauto ist eine Option die dafür sorgt dass das Gerät erst auf Anforderung eingebunden wird, sei es von Hand oder durch einen Ereignisdaemon.

Zu grub2
Zum einen habe ich auf der Grub2-Homepage gelesen dass die Benennung der Partitionen anders als bei Grub Legacy erfolgt.

Meine zweite Frage wäre dann: wie installierst Du Grub2? Du sprichst von einem Auswahlfenster, deswegen vermute ich ein grafisches Frontend.
Auf http://www.gnu.org/software/grub/manual/html_node/Installing-GRUB-using-grub_002dinstall.html ist das Programm "grub-install" auf der Kommandozeile aufzurufen.

Ein Diskettenbeispiel ist
Code:
But all the above examples assume that GRUB should use images under the root directory. If you want GRUB to use images under a directory other than the root directory, you need to specify the option --root-directory. The typical usage is that you create a GRUB boot floppy with a filesystem. Here is an example:

     # mke2fs /dev/fd0
     # mount -t ext2 /dev/fd0 /mnt
     # grub-install --root-directory=/mnt fd0
     # umount /mn


Grub2 wird noch heftig entwickelt, und mit logischen Partitionen scheint es noch Hürden zugeben:

Zitat:
To my own surprise there does seems to be a bug in Grub 2 which only effects logical partitions:

Code:

sudo grub-install -f /dev/sdaX

does not work if /dev/sdaX is mounted and /dev/sdaX is a logical partition. To make matters worse, grub-install exits with "Installation finished. No error reported." So there is no indication that something went wrong (other than the warning about "blocklist", but that warning is also displayed when "grub-install" was successful).


Trotzdem viel Erfolg!

vielen Dank, AndreasM, die Antwort war sehr hilfreich.

Die von mir angesprochene Sachlage beinhaltet eigentlich 2 Probleme:

1. Grub2:
Grub2 zählt die Partitionen wieder wie LILO, ist auf automatische Erkennung getrimmt und will unbedingt in den MBR, wie auch aus Deinem Link zu ersehen ist. Wohl deshalb ist wahrscheinlich eine Installation des Bootladers auf eine Diskette (wer hat denn noch so etwas außer mir?) nicht möglich.

Grub2 arbeitet ohne zu fragen mit 'uuid' für die Partitionen. Das muß kein Nachteil sein, solange man keine Partition löscht und neu anlegt: die uuid ist dann im Gegensatz zu sdax anders und es gibt Startschwierigkeiten.

Grub2 ist nun ohne Alternative in ubuntu 9.10 und folgende integriert, leider ohne dem Benutzer die Möglichkeit zu geben, auf die Art der Installation Einfluß zu nehmen. Nachträglich braucht man unzählige Befehle ( setup, install, upgrade und was weiß ich noch alles), um Grub2 den eigenen Wünschen anzupassen. Mit der Existenz anderer Bootlader rechnet Grub2 gar nicht erst. Es tut mir leid, ein wenig erinnert mich das an eine Firma aus Redmond.

2. Automount Diskette:
Auch ich brauche ja keine Diskette mehr. Interessant wäre nur gewesen den Bootlader auf Diskette zu installieren, wie das mal bei LILO möglich war. Mit der Zeile

Code:
/dev/fd0        /media/floppy0  auto    rw,users,auto,exec,utf8 0       0


kann ich nun auch die Diskette als Normal-User mounten. Das nutzt aber nichts für den bootlader, weil man das erst nach der Installation ändern kann. Auto-umount geht trotzdem nicht, der Disketteninhalt bleibt in Nautilus erhalten, auch wenn man die Diskette entnimmt. Was solls? Bei USB-Geräten klappt alles einwandfrei und das ist das Gute bei den neuen Distris.

Nun bin ich gespannt, was Ubuntu 10.04 LTS zu bieten hat.
