Zitat:
ich habe bereits mehrfach versucht das System neu drauf zuspielen, vergeblich.


Welches System hast Du mehrfach versucht aufzuspielen? Linux, also OpenSuSE. Oder wieder Windows?

Generell: höre zunächst auf herumzuinstallieren. Wenn die OpenSuSE-CD einen Live-Modus hat kannst Du darüber zunächst versuchen Deine Daten auf eine externe Festplatte zu sichern. Ansonsten ist für so etwas eine etwas andere Linux-Distribution zu empfehlen: Knoppix ( www.knoppix.org )

Zitat:
Nun habe ich die Trennung von "sda5" wieder aufgehoben, und versucht "sda1" wieder zu erstellen, was nicht perfekt möglich war, da OpenSuse kein NTFS System unterstützt, was heißt Windows ließ sich immernochnicht installieren.


Das ist so nicht richtig. Jedes aktuelle Linux kommt mit NTFS klar, lesend sowieso. ( Deswegen bietet sich eine Linux-Live-CD als Datenrettungssystem förmlich an: verschiedene Dateisysteme von DOS über Windows über MacOS über Unix bis OS2 ).
Nur wenn man Linux installiert wird in den Linux-Partitionen ( also die Partitionen in denen die Verzeichnisse der Standard-Verzeichnisbaums landen ) mit ext2, ext3, ext4 oder reiserfs als Dateisystem benutzt und ggf. formatiert.

Ich tippe bei Deinem Problem auf Fehlbedienung. Selbst wenn ein Installer mal abschmieren sollte beeinträchtigt das ein bestehendes System nicht ( sofern man die Windows-Partition nicht irrtümlich fprmatiert hat) , denn der Bootloader wird erst am Ende der Installation geschrieben, erkennt eine bestehende Windows-Partition und nimmt sie ins Auswahlmenü des Bootloaders auf.

Ich habe eine böse Vermutung:

Zitat:
"sda1" C: Boot ca. 139GB

Hast Du C: mal selber so genannt?

Im Linux-Verzeichnisbaum gibt es einen Ordner "/boot", in dem der Bootloader seine Konfiguration liegen hat. Ich würde tippen dass Du bei der Linuxinstallation und Partitionszuordnung

Zitat:
"sda1" C: Boot ca. 139GB

dem Systemverzeichnis "/boot" zugeordnet hast.

Und wenn aufgrund dieser Zuordnung Dein C:Laufwerk mit dem ext-Dateisystem formatiert wurde hast Du Dir Dein Windows plattgebügelt.

Aber das ist alles Vermutung. Genaues weiss man wenn Du eine Live-CD benutzt. Zumeinen kann man damit direkt ins Internet ( Chatten, Surfen, eMail ), und kann somit gleichzeitig auf seinem Rechner nach Fehlern suchen und im Internet Recherche bereiben. Wenn Du eine Live-CD hast können wir mal eine Zeit ausmachen, uns im IRC ( siehe Bild oben links ) treffen und mal Fehlersuche betreiben, evtl. auch mit Fernzugriff via ssh.

Für Interessierte:
Die Installationsroutine von OpenSuSE hatte nichts formatiert und keine Daten auf die Festplatte geschaufelt.
Bei der Sichtung mit fdisk haben wir festgestellt. dass das Boot-Flag auf sda2 statt auf sda1 gesetzt war.
Mit Setzen des Boot-Flags auf sda1 und Deaktivieren des Boot-Flags auf sda2 ( fdisk von Knoppix-Live-CD ) war das Problem behoben.

P.S: Hat mit Knoppix 6.2 ein neuer Bootloader Einzug gehalten? Es gibt ein für mich neues Auswahlmenü mit 14 Punkten.... www, shell, komplettes grafisches UI etc....
