hallo

Ich habe folgendes Problem. Auf meinem Webspace kann ich Ordner nicht mehr löschen, die über ein PHP Skript erstellt wurden. Z. B. von phpBB. Dieses Problem ist auch dem Support meines Webspaces bekannt, er schrieb folgendes.
Zitat:
Dateien die übr ein PHP Upload-Script auf den Server übertragen wurden, können per FTP nicht gelöscht werden.
Das gleiche Phänomen tritt auf bei Dateien und Verzeichnissen, die über andere PHP-Scripts erstellt wurden.
Grund für dieses Verhalten ist die Tatsache, dass der PHP-Interpreter als Servermodul des Apache Web-Servers ausgeführt wird.
Aufgrund dieser Tatsache werden Dateien und Verzeichnisse von PHP jeweils unter dem Account des Web-Servers erstellt.
Dermassen erstellte Dateien und Verzeichnisse können nicht per FTP gelöscht werden, weil die hierfür erforderlichen Zugriffsrechte fehlen.
Wir empfehlen Ihnen in diesem Fall folgendes:

* Löschen der Dateien/Verzeichnisse mit Hilfe eines PHP-Scripts
* Anfrage an den Support zur Löschung der Daten

Diese Tatsache, erscheint mir noch logisch, da ich per FTP so gut wie keine Rechte habe. Da ich nicht jedes mal, wenn ich gerade verschiedene Dinge ausprobieren möchte und dann den Ordner löschen will, den Support kontaktieren möchte, habe ich mich fürs PHP-Skript entschieden. Habe dann im Internet nach einem Skript gesucht, das Ordner rekursiv löscht. Folgendes habe ich gefunden:
Code:
<?php
function delete($file) {
  chmod($file,0777);
  if (is_dir($file)) {
    $handle=opendir($file);
    while($filename = readdir($handle)) {
      if ($filename != "." &amp;&amp; $filename != "..") {
      delete($file."/".$filename);
      }
    }
    closedir($handle);
    rmdir($file);
  }
  else {
    unlink($file);
  }
}
delete("cache");  //Ordername
?>


Habe dieses Skript auf meinen Webspace geladen. Ausgeführt, aber leider ist nicht passiert. Der Ordner ist immer noch gleich vorhanden wie vor her. Sieht jemand den Fehler?

So endlich geschafft, es sind alle Ordner weg, die ich nicht mehr haben will. Ich versuch einmal zu erklären wie es funktioniert hat. Oder was ihr machen könnt, wenn ihr das selbe Problem habt.

Zuerst muss man im FTP Client die Attribute aller Ordner und Unterordner auf 777 setzen. Danach nimmt man das Skript von http://aktuell.de.selfhtml.org/tippstricks/php/verzeichnisse/ (thx metoneca) und löscht den Hierarchisch höchsten. Dann geht man wieder in den FTP Client. Falls die Ordner immer noch nicht weg sind (waren sie bei mir nie ), dann probiert man sie wieder mit dem FTP Client zu löschen. Wenn danach immer noch Ordner da sind, einfach wieder die Attribute anzeigen lasssen und bei mir waren die Ordner dann weg (auch nicht immer). Am Schluss den Hierarchisch höchsten noch mit dem FTP Client löschen und weg isser. Falls nicht, könnt ihr probieren jeden einzelen Ordner mit dem PHP Skript zu löschen und dann wenn er noch da ist mit dem FTP Client löschen. Wenn er dann immer noch da ist Attribute anzeigen lassen, dann...

Verstanden? Ich auch nicht. Müsst einfach pröbeln .
