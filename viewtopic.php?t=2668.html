Mit PHP ein Shell Script ausführen (via Socket)

Ich musste mich vor geraumer Zeit mit der Frage auseinandersetzen: "Wie kann ich ein Shell Script mit PHP ausführen?". Es gibt sicherlich noch mehr Möglichkeiten um ein Shell Script mit PHP auszuführen. Mir gefiel diese Lösung aber besonders gut, weil man auch mit einem anderen Programm auf den Socket zugreifen kann, z.B. Telnet oder ein selbstgeschriebenes C-Programm. Als Beispiel habe ich ein Script erstellt, mit dem man einen Benutzer anlegen kann.

Mein System:
-Debian Etch
-Kernel 2.6.18


1.) Erstellen des PHP-Scriptes "index.php", welches die Daten an den Socket senden wird.

Code:

<?php
if ($_GET["action"] == "add")
{
  // Benutzernamen und Passwort über $_POST holen
  $username = $_POST["username"];
  $password = $_POST["password"];

  // Variable mypasswd wird mit in das Formular eingegeben, als "Sicherheitsabfrage"
  $mypasswd = $_POST["mypasswd"];

  // überprüfen ob Benutzername und Passwort Daten enthalten
  if ($username<>"" and $password<>"")
  {
   // Socket an Port 9222 öffnen
   $fp = fsockopen("localhost", 9222, $errno, $errstr, 30);

   // überprüfen ob der Socket geöffnet werden konnte
   if (!$fp)
   {
     // nein: Fehler ausgeben
     echo "$errstr ($errno)<br />\n";
   }
   else
   {
     // ja: Daten an Socket senden
     fputs($fp,"$mypasswd:$username:$password\n");

      // da wir bei einem Fehler eine Antwort vom Server erwarten, fangen wir diese ab
     $antwort=fgets($fp);

     // kein Fehler aufgetreten = keine Antwort
     if($antwort == "")
     {
       echo "Benutzer wurde erfolgreich angelegt!";
      }

     // Ausgabe der Meldung vom Server
     else
     {
       echo "Ein Fehler ist aufgetreten: $antwort";
     }

     // Trennen der Verbindung zum Socket
     fclose($fp);
    } 
  }

  // falls keine Daten in $username und $password enthalten: Fehler melden
  else
  {
    echo "Bitte Benutzernamen und Passwort eingeben!";
  }
}
?>

<html>
<head><title>Benutzer hinzuf&uuml;gen</title></head>
<body>
<hr>
<!--Formular zum eintragen der Daten -->
<form action="index.php?action=add" method="post">
Benutzername:<input type="text" name="username"><br>
Passwort:<input type="text" name="password"><br>
Admin-Passwort:<input type="text" name="mypasswd"><br>
<input type="submit" value="Senden">
</form>
</body>
</html>


Zur Erklärung der PHP-Funktion fsockopen(): Link


2.) Das Bashscript in "/root/bin/./adduser.sh" welches den Benutzer anlegen wird.

Man erstellt ein Verzeichnis für unser Batch Script, legt die adduser.sh darin an und macht sie zu einer ausführbaren Datei.
Code:

mkdir /root/bin
touch /root/bin/adduser.sh
chmod +x /root/bin/adduser.sh



Der Inhalt der adduser.sh:
Code:

#!/bin/bash

# Zeile die wir vom PHP-Script bekommen einlesen
read line

# erste Variable einlesen -> in $line steht jetzt $mypasswd:$username:$password.
# mit echo wird alles ausgegeben und mit cut -d: (Trennzeichen ":") -f1 (erste Spalte) erhalten wir den Inhalt von $mypasswd
AUTH=`echo $line | cut -d: -f1`
USRN=`echo $line | cut -d: -f2`
PWD=`echo $line | cut -d: -f3`

# da wir in USR unseren zukünftigen Benutzernamen stehen haben, in diesem aber keine Umlaute und Sonderzeichen erlaubt sind entfernen wir diese
USR=`echo $USRN | tr -d ' =_$%$[]{}()ÄÜÖäüö\-!' | tr [A_Z] [a_z]`

# Abfragen des Passwortes für das Anlegen
if [ "$AUTH" != "meintollespasswort" ];
  then
    echo "Passwort falsch!";

elif [ "$USR" == "root" ];
  then
    echo "Haettest du wohl gerne?";

# ... evtl. weitere Abfragen   

else
  useradd -m -d /home/"$USR" "$USR"
  echo "$USR":"$PWD" | chpasswd
fi


Das Zeichen ` ist ein "Backtick".


3.) Eintragen der Ports in "/etc/services":

Als nächstes wird festlegt, an welchem Port der Server "horchen" soll. In diesem Fall
hab ich den Port 9222 gewählt, da dieser nicht verwendet wird.
Code:

# Local services
adduser      9222/tcp   #benutzer anlegen


4.) Um den Service mit dem Bash Script zu verknüpfen

Dazu müssen wir folgende Zeile in "/etc/inetd.conf", der Konfigurationsdatei des Internet Service Daemon, eintragen:
Code:

#<service name> <socket type> <protocol> <flags>   <user>  <server path>     <arguments>
adduser         stream        tcp        nowait    root    /usr/sbin/tcpd    /root/bin/./adduser.sh


Internet Service Daemon neustarten:
Code:

/etc/init.d/openbsd-inetd restart


5.) Der Testlauf

Internetbrowser öffnen, URL zum Script eingeben, Formular ausfüllen (In meinem Beispiel ist das Admin-Passwort: meintollespasswort) und senden.


6.) Troubleshooting

-Ob überhaupt eine Verbindung zum Socket möglich ist, lässt sich damit herausfinden:
Code:

<?php
  $fp = fsockopen("localhost", 9222, $errno, $errstr, 30);

  if (!$fp)
  {
   echo "$errstr ($errno)<br />\n";
  }
  else
  {
    echo "Verbindung hergestellt!"
  }
?>


-Testen des Bash-Scriptes
Code:

/root/bin/./adduser.sh


Dann erwartet das Script eine Eingabe:

meintollespasswort:neuernutzer:passwort

Wenn alles geklappt sollten keine Fehlermeldungen ausgegeben werden und der Benutzer sollte angelegt sein.



Für Kritik und Verbesserungsvorschläge bin ich natürlich offen und wünsche euch nun viel Spaß beim probieren.
