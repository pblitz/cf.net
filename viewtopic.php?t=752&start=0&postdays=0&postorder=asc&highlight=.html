nein, du schreibst im internet auch nicht ftp://ftp.ip sonder ftp://ftp.domain.tld.

das ftp.domain.tld (also z.b. ftp.computer-forum.net) ist eine Alias für eine bestimmte IP Adresse (Stichwort DNS), das dann zu einer IP (in diesem fall 83.133.52.244) auflöst.

das du ftp.computer-forum.net hat nicht damit zu tun, das du einen ftp server benutzt. es steht einfach nur für einen host bzw. eine Service, der unter der domain computer-forum.net genutzt wird
Beispiel:
http://ftp.leo.org
ftp://ftp.leo.org (beides der gleiche server, beides die gleiche daten, aber beides nen anderes Protokoll)

Einfach gesagt:

ftp:// und http:// zeigen den Port an, also die Tür in einem Haus

www.computer-forum.net ist eine Adresse


Im "Land" (.net ist kein Land, aber egal ) in der Stadt computer-forum (jaja ...) das Haus www

Aber das alles ist eher virtuell und dient zum finden von IP Adressen. Eine IP Adresse kann aber viele Domains haben.

Die gleiche IP kann problemlos folgende Domains und mehr haben:

www.test.de
ftp.test.de
heiko.test.de

Alles vor test.de ist einfach aus Gewohnheit da und rein optional, http://test.de ist je nach Domainkonfiguration problemlos möglich.

Aber es ist halt üblich geworden den Dienst als Teil der Domain zu schreiben, notwendig ist es aber nicht. Man kann den FTP auch z.B. unter ftp://dasistkeinftp.test.de betreiben, das ist FTP egal solange auf dieser Domain/der IP dahinter ein FTP Server läuft

Übrigens ist das Level des Domainsystems (afaik) unbegrenzt (bzw. 255 Zeichen gesamt), es kann also auch wie folgt aussehen:

http://ich.will.eine.lange.domain.fuer.einen.test.de
