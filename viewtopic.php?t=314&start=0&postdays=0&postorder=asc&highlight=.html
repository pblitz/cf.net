Hi

Folgender Code ist jetzt aus dem Kopf geschrieben, sollte aber ok sein.

WICHTIG WICHTIG WICHTIG:
NIE NIE NIE eine übergebene Variable direkt ansprechern (index.php?test=hallo als $test). Das ist extrem unsicher und macht später enorme Probleme. IMMER über $_REQUEST bzw. $_GET und $_POST gehen.
Am besten auf dem Server dann register_globals = off setzen (in der php.ini)

Code:

<?php

$d = dir("verzeichniss/");

while (false !== ($entry = $d->read()))
{
  if ($entry == '..' || $entry == '.')
    continue; // . und .. brauchen wir nicht

  $parts = explode('-',$entry); // trennen den Dateinamne am - auf

  if ($parts[0] == $_GET['titel']) // ist der name vor dem - das gesuchte?
    echo '<img src="'.$entry.'" border="0" /><br />';
}

?>


Anmerkung: bitte nie mehr <br> schreiben, Tags immer schliesen!
Und: Variablen in einem "" in PHP direkt zu schreiben ist auch BÖSE: Das ist sehr schwer zu lesen, und verlangsamt PHP. Soetwas immer in '' packen, und den string an den entsprechenden Stellen unterbrechen. Damit spart man sich auch die ganzen \ die sonst gebraucht werden.

Das Problem bei PHP ist, das es viel zu ungenau selbst prüfst. Du soltest daher deinen Code immer so aufbauen, das jede externe Eingabe als extrem gefährlich angesehen wird.

Daher lade ich in meinem Beispiel auch die Liste der vorhandenen Dateien, und vergleiche von dort aus die Namen mit der Übergabe. Da kann eigentlich nichts schiefgehen. Wenn du aber eine übergabe als Dateinamen nimmst, geht es meistens schief.

Das selbe gilt für Datenbanken. Wer direkt die MySQL und co. Funktionen von PHP nimmt, hat fast schon verloren
Besser ist es z.B. AdoDB zu nehmen, und deren Quote() funktion. Das hilft dann schon wesentlich.

Aber natürlich sollten auch alle Integer/Zahlen in PHP immer auch so behandelt werden. Leider ist das in PHP aber von Start weg nicht so. Ein String und ein Int sind für PHP erst bei einer Rechnung unterschiedlich, außer du wandelst es um / lässt es prüfen.

Das kann alles auch sehr schnell peinlich werden.
Wenn man z.B. einen "PC Spezialisten" wie bei der Sat 1 Sendung Planetopia holt, der keine Ahnung hat, und dessen Homepage wenig später durch solch einfache Tricks als offen wie ein Buch erkannt wird, ist das schon sehr sehr peinlich
