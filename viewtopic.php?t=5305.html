Hallo zusammen,

ich habe hier ein Problem mit der Funktion des cmd unter Windows7 bzw. Windows Server 2008 R2.

In der Batchdatei ist folgende Zeile zu finden:

C:\Programme\Pk1.exe d:\TEMP\Aufbereiten.ctl

Das Programm PK1 liest Informationen aus einer Textdatei aus und führt die darin enthaltenen Anweisungen aus ( Ist ein COBOL-artiger Dialekt ) Die besagte Text/Skriptdatei ist natürlich Aufbereiten.ctl

Unter WindowsXP funktioniert der Aufruf. Das Programm startet und arbeitet die in der Skriptdatei enthaltenen Anweisungen ab.
Unter Windows7 hingegen funktioniert das nicht.
Unter Windows7 startet das Programm PK1 zwar, fragt dann aber in einem Suchdialog nach der Skriptdatei - als würde der Parameter in der Batchdatei ignoriert.
Kann jemand von Euch mir das erklären?
