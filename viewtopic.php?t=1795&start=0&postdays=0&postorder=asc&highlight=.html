interessantes thema, hab grad mal ein bisschen wikipedia gewälzt.
Deckt sich mit dem was ich noch wusste:

SGML ist eine Markup-Sprache (sehr breit gefasst, du gibst ihm ne DTD und er kann damit deine Mutter ihre Küche darstellen)
XML ist ein dialekt von SGML (ziemlich direkter) allerdings etwas abgespeckt. Daher auch das Konzept von DTDs, was HTML in der Urform nicht kennt

HTML selber war wohl lange zeit auch eine Ableitung von SGML, sollte dann aber mal eine Anwendung (also durch SGML beschreibbar) werden. Wurde es aber nie so wirklich, und heute ist HTML ja eh eine abwandlung von XML (also wiederrum ein Dialekt von SGML)

Daher, ja, mann kann wohl in SGML HTML definieren. braucht man aber nicht und macht auch keiner.
Und beides sind Metasprachen (HTML zumindestens theoretisch).
Sie definieren die Metainformationen zu Objekten. Ich überlege grad was du mit Objektsprache meinst?

Und klar, wenn du dich an den SGML syntax hälst, kannst du damit eig. alles machen. Es ist ein XML, das mehr kann als XML, so gesehen. Der Syntax ist halt etwas zu kopmlex, als das es verbreitung gefunden hätte. Und ich glaube nicht, das Standard webbrowser SGML noch rendern könnten. Wobei, mozilla würde ich das zutrauen ;D

[edit]:http://www.w3.org/TR/NOTE-sgml-xml-971215.html ist ganz interessant, XML vs. SGML [/edit]
