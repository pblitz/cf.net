Reguläre Ausdrücke.
Bei sed bin ich mir aber nicht sicher ob das noch funktioniert wenn sich das p-Tag über mehrere Zeilen erstreckt, oder ob sed zeilenweise orientiert arbeitet.

Ansonsten würde ich auf die Schnelle versuchen:

Zitat:
<p[^>]+>


Suche
ein öffnende Spitze Klammer (<)
gefolgt von einem p (p)
gefolgt von mindestens einem (oder beliebig vielen) Zeichen ausser der schliessenden Klammer ( [^>]+)
gefolgt von einer schliessenden Kammer (>)

Wenn dieser Reguläre Ausdruck gefunden wird ersetze alles durch <p>

Wenn Du über mehrere Zeilen suchst musst Du ggf. noch einen Modifikator mitgeben


In Perl-Notation sieht das dann so aus. Das fettgeschriebene wird gesucht und durch das kursiv geschriebene ersetzt:

s/<p[^>]+>/<p>/gsi (g = global, ersetze alles, i = ignoriere Gross/Klein, s = finde auch einen Zeilenumbruch wenn "beliebieges Zeichen" gefragt ist

Code:
#! /usr/bin/perl
use strict;
use warnings;

my $Dateiname = "c:/Dateien/wennes Windows ist/Datei.html";

open (EINGABE,"<$dateiname");                 #öffne Datei zum Lesen
open (AUSGABE,">$dateiname.neu.html");  #öffne Datei zum Schreiben

my $text = <EINGABE>;                   # zu lesende Datei komplett in eine Variable einlesen
  $text =~ s/<p[^>]+>/<p>/gsi;      # Variable bearbeiten
 print AUSGABEDATEI $text;             # wegschreiben

close EINGABE;
close AUSGABE;

Code:
#! D:\Perl\bin

use warnings;
use strict;
use Cwd;
use File::Find;

my $Pfad = cwd;
undef $/;

find(\&htmlsuche, "$Pfad");


sub htmlsuche {
   my $Datei = $_;
   print "$Datei\n";
   if ( (-T $Datei ) and ($Datei =~ /.+\.html?$/i) ) {
      open IN,  "$Datei";
        open OUT, ">$Datei.new";       
        while ( <IN> ) {
            s/&nbsp;<\/P>.<P ALIGN=JUSTIFY>//gi;
            print OUT ;
        }
   }
   close IN;
    close OUT;
}


Hallo schulle, kopier Dir den Code in eine Textdatei und speichere die als Perl-Datei ab.
Unter Windows reicht dazu die Endung ".pl"
Wenn Du mit Unix/BSD/Linux arbeitest musst Du die erste Zeile ändern, denn die wird unter Unixsystemen ausgewertet. Du musst also ggf. den Pfad zu Deinem Perl-Interpreter eintragen.

Den RegAusdruck erkläre ich Dir ggf. nochmal:

Code:
use warnings;
use strict;


sind Pragmas die ich standardmässig immer drin habe. Sie erzwingen eine etwas sauberere Skriptprogrammierung.

Code:
use Cwd;
use File::Find;


Das sind zwei Module die ich hier nutze und deren Befehle/Variablen hier importiert werden. Cwd ist nützlich wenn Du den aktuellen Pfad einlesen willst. Zu Beginn des Skripts ist es der Pfad aus dem heraus das Skript gestartet wird. Deswegen mein Tipp das Skript in das Verzeichnis zu kopieren unter dem Deine HTML-Dateien liegen und von dort anzustossen.
File::Find dient dem rekusiven Durchsuchen von einem Verzeichnis (oder auch mehreren Verzeichnissen )

Code:
undef $/;
$/ ist eine Spezialvariable. Sie beinhaltet das Zeichen bis zu dem eine Datei gelesen werden soll. Standardmässig ist das das Newline-Zeichen (\n). Eine Datei wird also standardmässig immer zeilenweise gelesen.
Weil ich hier aber die Datei in Gänzer einlesen will setze ich die variable als undefiniert.

Code:
my $Pfad = cwd;
liest den aktuellen Pfad aus und speichert ihn in einer Variablen. Ich brauche ihn in der nächsten Zeile:

Code:
find(\&htmlsuche, "$Pfad");


Der Befehl "find" aus dem ( Modul File::Find ) erhält 2 Argumente mit auf den Weg: eine Subroutine ( &htmlsuche ) die für jede gefundene Datei/Verzeichnis aufgerufen werden soll, und eine Liste von Startverzeichnissen die rekursiv durchsucht werden sollen. Hier habe ich keine echte Liste, sondern nur eine einzelne skalare Variable: das mit cwd ausgelesen Startverzeichnis ( $Pfad ).

Code:
sub htmlsuche {
   my $Datei = $_;
   print "$Datei\n";
   if ( (-T $Datei ) and ($Datei =~ /.+\.html?$/i) ) {
      open IN,  "$Datei";
        open OUT, ">$Datei.new";       
        while ( <IN> ) {;
            s/&nbsp;<\/P>\s*\n\s*<P ALIGN=JUSTIFY>//gi;
            print OUT ;
        }
   }
   close IN;
    close OUT;
}


macht die eigentliche Arbeit. File::Find unterscheidet nicht zwischen Datei und Verzeichnis. Alles wird besucht und ggf. untersucht.
Deswegen:

Code:
my $Datei = $_;
Was File::Find gerade untersucht speichere ich erstmal in eine Variable. Nun, erstmal ist das nur der Name. Das print darunter ist überflüssig: es gibt den Inhalt der Variablen auf der Konsole aus. Nur interessant wenn man sehen will was gerade bearbeitet wird. Lösch das ruhig.

Code:
if ( (-T $Datei ) and ($Datei =~ /.+\.html?$/i) ) {
...
}

Eine If-Schleife die durchlaufen wird, wenn es/das eine Textdatei ist ( -T $Datei ) UND mit .htm(l) endet.
Code:
       
        open IN,  "$Datei";
        open OUT, ">$Datei.new";       
        while ( <IN> ) {
            s/&nbsp;<\/P>\s*\n\s*<P ALIGN=JUSTIFY>//gi;
        print OUT ;
        }


Die Datei wird zum Lesen über das Dateihandle IN geöffnet.
Eine neue Datei "$Datei.new" wird über das Dateihandle OUT zu schreiben geöffnet.
Das Dateihandle IN wird komplett eingelesen und der Inhalt untersucht.
s/&nbsp;<\/P>\s*\n\s*<P ALIGN=JUSTIFY>//gi;

Das erste s ist der Ersetzungsoperator. dadurch wird der reguläre Ausdruck zwischen dem 1ten und 2ten Slash durch den Ausdruck zwischen dem 2ten und 3ten Slash ersetzt.

s/suche das/ersetze dies/;

Hinter dem dritten Slash finden sich Modifier: i ignoriert GrossKlein, g sucht wiederholt bis zum Ende (sonst würde nur der erste Treffer ersetzt werden)
Ich suche nach
Zitat:
&nbsp;<\/P>\s*\n\s*<P ALIGN=JUSTIFY>
und ersetze das durch nichts .

\s* sind Leerzeichen die sich eventuell vor oder hinter dem Newline-Zeichen ( \n ) befinden könnten. Deswegen habe ich die noch ergänzt.
<\/P> sucht nach dem Text "</P>". Weil der Slash eben eine Sonderbedeutung hat muss ich ihn, wenn ich ihn als Text suche, escapen. Das geschieht mit dem vorangehende Backslash.
