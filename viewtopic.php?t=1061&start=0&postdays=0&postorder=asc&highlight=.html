Open Source wird Software genannt, die Du im Quellcode bekommen kannst. Also so, wie sie programmiert wurde. Du kannst dann, wenn Du es denn kannst, die Software sogar verändern und ggf. verbessern.
Sehr viele bekannte Serveranwendungen sind Open-Source-Software, auch manche Officepakete. Apache (Webserver), Sendmail, Postfix (Mailserver), MySQL, PostgreSQL (Datenbanken), OpenOffice (eine komplette Office-Suite), Perl und php (Skriptprogrammiersprachen. Besonders mächtig in Verbindung mit -> Apache und -> MySQL), Thunderbird, Firefox, ), Linux (Betriebssystem. So etwas ähnliches wie Windows.Nur flexibler) phpBB, MediaWiki, TWiki....und und und.
In den allermeisten Fällen steht diese Software gratis zum Download zur Verfügung - auch für Firmen.

Hintergrund ist der Gedanke, das quelloffene Softwareimmer besser kontrolliert und somit auf Fehler überprüft werden kann... und auch schneller Verbesserungen erfährt. Eben weil nicht nur eine Handvoll Programmierer in einer Firma den Quellcode kennen, sondern Tausende.

Der Code wird in einen Bytecode umgewandelt, also quasi in einen Pseudo assembler code.

Natürlich ist es unmöglich den code so zu verändern das er NIE verstanden wird, das wäre ja schon deswegen unmöglich weil der PC ihn ja verstehen muss. Aber man kann ihn so umbauen das er nur für eine Maschiene wert hat und von einem Menschen nur mit sehr großer Mühe verstanden werden kann.

Beispiel: Alle unnützen umbrüche und leerzeichen raus, Klassen und variablen umbenennen (komplex und unaussprechlich), kommentare entfernen, schleifen durch sprünge ersetzen (sowas geht in php afaik nicht) und dann in einen optimierten Bytecode umwandeln. Sollte doch noch jemand den zurückwandeln können (mit weiteren verlusten) ist der code nur sehr schwer zu gebrauchen.

Das selbe problem gibt es auch mit compiliertem C++ Code. Man kann ihn in gewisser weise zurückverwandeln, aber das Produkt ist nicht annhäernd das was es mal war.

Oft ist der "Container" in dem der Bytecode liegt selbst auch nochmal gesichert.
