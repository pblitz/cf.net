Hallo Leute,

dass viele Online-Shops besonders gesichert sein müssen, ist mir klar. Ab und zu kann ich beobachten, dass ein https aus dem normalen http in der Adresszeile wird. Ich habe mich dazu schon ein bisschen belesen und bei Wikipedia gefunden, dass es sich hierbei um ein Übertragungsprotokoll handelt. Alle wichtigen Daten werden also gesichert übermittelt und so geschützt.

Etwas verwirrt bin ich dennoch, denn auf http://www.globalsign.de/ssl-informationszentrum/was-ist-ssl.html wird erklärt, dass das Schloss-Symbol eine wichtige Bedeutung hat.. Wenn man eine sichere Übertragung hat, soll das Schloss angezeigt werden. Wenn ich mich bei Amazon oder eBay aber einlogge, kommt nicht dieses Symbol. Soll das heißen, dass bisher alle Verbindung unsicher waren? Neben HTTPS wid auch SSL auf der Internetseite genannt. Das ist auch ein Sicherheitsprotokoll, aber soll das irgendwie anders als https sein?

Dann wollte ich noch fragen, was diese Identitätsdaten sein sollen. Diese Anzeige ist direkt mit dem Icon der Website verbunden. Das erscheint auch nicht einfach so. Sobald man sich irgendwo einloggt, wird das Favicon vergrößert und die Anzeige ist zu sehen. Es erscheint dann auch die Meldung, dass ich im Augenblick mit www.facebook.com, www.google.de, www.ebay.de, www.neu.de oder www.amazon.de verbunden bin. Hat das auch etwas mit der sicheren Übertragung der Daten zu tun?

Zum Schluss habe ich noch eine Frage wegen eines Add-ons für den Firefox. Es betrifft das Plugin HTTPS Everywhere Firefox-Webbrowser von http://www.chip.de/downloads/HTTPS-Everywhere-fuer-Firefox_43521096.html. Auf Chip steht, dass jede Website anonym angesurft werden kann. Stimmt das wirklich? Ich dachte eigentlich, dass ich immer zurückverfolgbar bin. Welchen Zweck hat das Plugin? Welchen Effekt oder Nutzen hat das dann eigentlich für mich? Auf der offiziellen Website von Firefox https://addons.mozilla.org/de/firefox/ ist das Add-on gar nicht aufgelistet. Ist das Add-on nicht mehr aktuell?

Ihr kennt euch bestimmt besser aus als ich.

Danke und bis dann

1)
Ob das Schlosssymbol angezeigt wird, hängt vom verwendeten Browser ab (in dem Beispiel auf der verlinkten Seite ist das der Internet Explorer). Sobald in der Adresszeile https://* steht, wird die Webseite verschlüsselt übertragen - unabhängig davon, ob das Schloss angezeigt wird. Wenn du links auf das favicon neben der Adresse der Webseite im Browser klickst, zeigt der Brower dir nochmal explizit an, ob und wie die Webseite verschlüsselt wurde.

Meines Wissens optimieren einige Unternehmen ihre Anwendungen, indem sie auch nach dem Login nur die wirklich sensiblen Daten verschlüsseln. Wenn du also auf eBay surfst und nach Produkten suchst, wird nicht unbedingt verschlüsselt - gehst du allerdings auf dein Profil oder loggst dich ein, sollte das geschehen

Wie du Wikipedia entnehmen kannst, liegt SSL im Protokollstapel unter HTTPS - HTTPS setzt also auf SSL auf, das sind keine gleichwertigen Protokolle.

2) Das heißt, dass die Identität des Website-Betreibers (der Webseite) geprüft wurde und du garantiert mit einem Server verbunden bist, von dem du etwas willst. Das wird dann wichtig, wenn jemand versucht, sensible Daten abzufangen und du also deine Daten an jemanden schickst, der sie gar nicht bekommen sollte.
Die Identität wird üblicherweise nur geprüft/angegeben, wenn sowieso Verschlüsselung verwendet wird.

3)Das Plugin scheint für die Anonymisierung TOR zu verwenden. Das ist immer dann komplett anonym, wenn dein PC wirklich überhaupt nichtsmehr direkt selbst nachlädt. Nachteil: interaktive Inhalte wie zb. Flash/Videos funktionieren meine ich mit TOR nicht oder nur sehr schlecht. Außerdem gibt es spezielle Header, um die Benutzung eines Proxies durch den Client festzustellen - keine Ahnung, ob TOR sowas ausfiltert.

Das Plugin benutzt automatisch HTTPS-Verschlüssung, sofern sie vom Server angeboten wird und schon Regeln für diese Seite definiert sind. ("HTTPS Everywhere can protect you only when you're using sites that support HTTPS and for which HTTPS Everywhere includes rules.") . Also surfst du bei unterstützten Seiten damit genau so sicher, wie der Betreiber es zulässt und wie gut und vollständig die Regeln definiert sind.
=> Welche Webseiten genau erfasst sind, siehst du auf der Webseite
=> für alle anderen wird es dir nach meinem bisherigen Verständnis nichts bringen
