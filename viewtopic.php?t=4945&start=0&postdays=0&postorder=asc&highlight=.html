Ja. Kein Grund zu schreien. Ich habe die Fragezeichen aus aus schmidtie's Zitat entfernt - aus dem von mir erwähnten Grund.

Wie gesagt: Hier bist Du im Bereich der Schnittstellenprogrammierung.
Ich möchte wetten das das Codetechnisch nur ein paar kurze Zeilen sind.

Mein erster Ansatz, um das Ganze auszuprobieren, wäre Perl. Es gibt ein Perl-Modul ( http://search.cpan.org/~scott/Device-ParallelPort-1.00/lib/Device/ParallelPort.pm ) Das übernimmt recht komfortabel Ansteuern und Auslesen des Parallelport .
Eine Windows-Portierung von Perl findest Du z.B. bei ActiveState, http://www.activestate.com/. Perl Ist OpenSoure und kostet Dich nichts.

Ausserdem müsstest Du das erwähnte Perl-Modul nachinstallieren,aber auch das ist mit dem ActiveState-Perl-Package-Manager kein Problem.

Ein Perlscript,das Endlos läuft, sähe dann z.B. so aus

Code:
#! c:\Perl

use strict;
use warnings;
use Device::ParallelPort ;

# Ein Parallelport-Objekt erzeugen, und zur Sicherheit Bit Nr. 3 auf 0 setzen
my $port = Device::ParallelPort->new();
$port->set_bit(3,0);

# Hier geht es in eine Endlosschleife
 while(1) {
     
     #Signal an Bit3 des ParPorts auslesen
     my $signal = $port->get_bit(3) ;
       
     # Wenn das Bit wen Wert "1" hat,dann mache was...
     #z.B. eine Sounddatei abspielen
     if $signal = 1 {
        System("spiel mir das Lied vom Tod, womit auch immer");
        #Nach dem Abspielen das Bit zurücksetzen
        $port->set_bit(3,0);
      }
}
