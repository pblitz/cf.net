Servus,

Nachdem meine Seagate Dockstar an ihre Grenzen stieß und der Raspberry Pi 4 Wochen nach Bestelldatum immer noch nicht da ist, habe ich mich nach etwas anderem umgesehen. Meine Wahl fiel auf den Trim-Slice von CompuLab.

Technische Daten kann jeder selber nachschauen ... http://www.trimslice.com

Ich habe den Trim-Slice H Diskless, nachgerüstet mit einer 16GB µSD-Karte (Class 10) und einer 64GB SSD.

Blödsinn wie Unboxing Video erspare ich der Allgemeinheit, das Teil kommt mit Netzteil, Adapter für RS232 und Audio sowie Bluetooth-Dongel (der aber nur je nach Modell).

Einsatzzweck soll wie auch schon bei der Dockstar eine Art Mini-Server für die Wohnstube sein - unter anderem Router / Pulseaudio-Server / Scan-Server / Druck-Server / VPN-Client (in Verbindung mit Routing) / VDR und was mir sonst noch so einfällt ...

Doku in aller Kürze / Erfahrungen meinerseits in den kommenden Posts meinerseits ... das wird kein Howto für Anfänger - ich werde mich hauptsächlich auf die Sachen mit Stolperpotential (in meinen Augen) beschränken ...

Das eigentliche System befindet sich bei mir auf der µSD.

Da der Bootloader uboot zur Zeit nur mit ext2 umgehen kann und ext4 doch einen Tick schneller ist, habe ich mir auf der µSD eine 100MB große ext2 Boot-Partition angelegt, der Rest als Root-Dateisystem ist mit ext4 formatiert.

Code:

mkfs.ext2 /dev/sde1
mkfs.ext4 /dev/sde2


... rootfs mounten, Mountpoint boot anlegen und die boot-Partition dahin mounten ...

Code:

mount /dev/sde2 /media/usb
mkdir /media/usb/boot
mount /dev/sde1 /media/usb/boot


... fertiges Root-Dateisystem laden und auf die µSD entpacken

Code:

cd /media/usb
wget http://archlinuxarm.org/os/ArchLinuxARM-trimslice-latest.tar.gz
tar -xzf ArchLinuxARM-trimslice-latest.tar.gz


... jetzt muss für den uboot Bootloader die Datei boot.scr erstellt werden - dafür eine für mein Setup gültige Vorlage namens bootcmd-file.txt (2 Zeilen: setenv... & mmc ...), "mmcblk0p1" habe ich wegen meiner Root-Partition nach p2 abgeändert, das die Datei uImage liegt auf der Bootpartition im Hauptverzeichnis ...


für µSD-Karte
Code:

setenv bootargs 'root=/dev/mmcblk0p2 rw rootwait console=tty1 console=ttyS0,115200n8 mem=384M@0M mem=512M@512M nvmem=128M@384M vmalloc=248M nohdparm noinitrd init=/sbin/init rootwait loglevel=8 video=tegrafb'
mmc init 1 && ext2load mmc 1 4080000 /uImage && bootm 4080000



für SD-Karte
Code:

setenv bootargs 'root=/dev/mmcblk0p2 rw rootwait console=tty1 console=ttyS0,115200n8 mem=384M@0M mem=512M@512M nvmem=128M@384M vmalloc=248M nohdparm noinitrd init=/sbin/init rootwait loglevel=8 video=tegrafb'
ext2load mmc 0:1 4080000 /uImage && bootm 4080000


... das Erstellen der boot.scr passiert mit dem Tool "mkimage" (uboot-mkimage) ...

Code:

mkimage -A arm -O linux -T script -C none -a 0 -e 0 -n "Trim Slice µSD Boot" -d /media/usb/boot/bootcmd-file.txt /media/usb/boot/boot.scr


... jetzt die µSD in den Slot der Trim-Slice, RS-232 Adapter dran, diesen mittels Nullmodem-Kabel an PC oder Notebook und mit einem Terminal-Programm seiner Wahl (hier picocom) dem Bootvorgang zusehen - 115200-N-8 ...

Code:
picocom -b 115200 /dev/ttyS0

pacman -S core/linux-trimslice core/linux-headers-trimslice

[/edit]

Der aktuell über die Repositories verfügbare Kernel ist zum einen nicht mehr ganz aktuell (2.6) und es fehlt das TUN-Device, welches ich für OpenVPN benötige - also wieder eine Baustelle.

Mit den Quellen, die direkt im Trim Slice Wiki angeboten werden, hatte ich ein paar Probleme, außerdem hätte das Bauen des Kernels über den einfachen "Dreisatz" sich nicht ins Paketsystem eingefügt - durch Zufall bin ich beim Suchen nach Firmware-Dateien auf die Seite http://myplugbox.com/.trimslice/linux-trimslice/ gestoßen - da gibt es alles, was man braucht um ein Paket mit makepkg zu erstellen.

... die benötigten Tools installieren ...

Code:
pacman -S base-devel


... ein Verzeichnis zum Bauen des Kernels anlegen, dorthin wechseln und dann das PKGBUILD, einige Patch- und Firmwaredateien und die Quellen selbst herunterladen ...

Code:

wget http://myplugbox.com/.trimslice/linux-trimslice/PKGBUILD

wget http://myplugbox.com/.trimslice/linux-trimslice/linux-trimslice.install

wget http://myplugbox.com/.trimslice/linux-trimslice/armdma.patch

wget http://myplugbox.com/.trimslice/linux-trimslice/build-oops.diff

wget http://myplugbox.com/.trimslice/linux-trimslice/change-default-console-loglevel.patch

wget http://myplugbox.com/.trimslice/linux-trimslice/config


... in der Datei /etc/makepkg.conf die Anzahl der Jobs erhöhen - wir haben ja einen Dualcore ...

Code:
MAKEFLAGS="-j3"


makepkg ausführen - ggf. bricht es beim ersten Versuch ab, weil Abhängigkeiten fehlen - diese dann einfach mit pacman nachinstallieren...

... ich habe in der Konfig für den Kernel das Modul "Universal TUN/TAP" und Unterstützung für iptables raw table hinzugefügt ...

wenn makepkg fertig ist erhält man zwei Dateien - einmal linux selbst und einmal die passenden Header - das installiert man mittels pacman -U, anschließend Reboot mit dem neuen Kernel

[edit]

da bei Arch Linux das Verzeichnis /lib nach /usr/lib wandert gibt es hier Schwierigkeiten, die ich noch lösen muss

[/edit]
