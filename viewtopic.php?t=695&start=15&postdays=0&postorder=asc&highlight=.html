Ah o.k.! Gut zu wissen!

Habe nun auch den Laptop als Router für den PDA erfolgreich eingeichtet.

Die verdammte Fehlermeldung von iptables (vom Anfang dieses Postings) ist nicht unwichtig. Hat bei meinem Laptop die Funktion gehindert.

Lösung:
"CONFIG_MODVERSIONS=y" macht in Zusammenhang mit IPTABLES ordentlich probleme! Nach Abschaltung konnten endlich iptables-module geladen werden.

Also jetzt mal in Kurzform:
Code:

Im Kernel deaktivieren:

Loadable module support-->
Set version information on all module symbols [ ]

im Kernel aktivieren:
Networking options-->
Packet Socket[M]
Network Packet Filtering[*]
Socket Filtering[*]
--->"Netfilter Configuration"
[i]fast alles aktivieren[/i]

[i]evtl. dnmasq installieren[/i]

Routing aktivieren:
sudo /sbin/iptables -F
sudo /sbin/iptables -X
sudo /sbin/iptables -t nat -F
sudo /sbin/iptables -P INPUT ACCEPT
sudo /sbin/iptables -P FORWARD ACCEPT
sudo /sbin/iptables -P OUTPUT ACCEPT
sudo /sbin/iptables -t nat -A POSTROUTING -s 192.168.131.201 -j MASQUERADE

echo 1 > /proc/sys/net/ipv4/ip_forward

Am PDA muss nichts eingestellt werden (im ggs. zu den veralteten Anleitungen im Netz)


Das ganze läuft mit einem Yakumo Delta 300. Dazu musste das ipaq-modul "ge"patched werden.

Höchstwahrscheinlich kann man sich einige Schritte einsparen, evtl. habe ich auch etwas vergessen.
