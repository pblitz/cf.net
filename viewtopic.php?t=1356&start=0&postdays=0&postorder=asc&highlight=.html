Huhu.
Ich versuche grade, in einem Bash-Script einem Befehl gewisse Parameter zu übergeben, die vorher in einer Variablen definiert worden sind.
Soweit kein Problem, aber wenn danach zusätzlich noch ein anderer Befehl ausgeführt werden soll, der ebenfalls in dieser Variable aufgelistet wird, so wird dieser nicht als eigenständiger Befehl, sondern als Parameter des ersten Befehls betrachtet.

Vereinfachtes Beispiel:
Code:
echo hallo; echo blub

gibt aus
Code:
hallo
blub


Folgendes habe ich vor:
Code:
var="; echo blub"
echo hallo $var

Es soll die gleiche Ausgabe wie oben erscheinen, aber die Ausgabe sieht natürlich so aus:
Code:
hallo; echo blub


Mehrere einzelne Parameter lassen sich ja sauber voneinander trennen, wenn man die Variable als Array benutzt, aber das bringt mich hier nicht weiter.
Desweiteren kann ich schlecht mit $(befehl) oder Backticks arbeiten, weil der Befehl erst bei der Auswertung, aber nicht bei der Zuweisung ausgeführt werden darf.
Der eigentliche Haken ist einfach, daß das ';' als normales Zeichen und nicht als Befehlstrenner betrachtet wird.

Das hier ist jetzt eine Ausnahme, wo ein zusätzlicher Befehl nötig ist. In den meisten Fällen reicht die Variable als Parameterliste aus. Deshalb habe ich mich bisher davor gesträubt eine zusätzliche Funktion zu schreiben, in der sich die einzelnen Befehle auflisten lassen, weil das den Rest unnötig kompliziert machen würde.

Jemand eine Idee?

Jau, stimmt. Das ist zwar auch ein bißchen von hinten durch die Brust in's Auge, aber mit einer extra Datei geht's.

Das ganze sieht dann jetzt in etwa so aus (um bei obigem Beispiel zu bleiben):
Code:

echo "echo hallo"$var > $tempfile
. $tempfile

Richtig, das `echo blub` wird allerdings direkt bei der Zuweisung an VAR ausgeführt und die Ausgabe dieses echo-Befehls (das 'blub') in VAR gespeichert.

Bei echo funktioniert das ja noch, wenn Zuweisung und Ausgabe direkt nacheinander stehen.
Aber wenn es sich bei den Befehlen z.B. um bestimmte make targets handelt, die nacheinander ausgeführt werden müssen, geht das nicht, weil dann die Reihenfolge nicht mehr stimmt. Außerdem würde auch eine bedingte Ausführung unter den Tisch fallen, wenn die Befehle per && getrennt wären.

Die ganze Aktion ist etwas komplizierter, mir ist auch leider kein besseres Beispiel eingefallen.

Das 'echo Hallo' soll vor 'echo blub' ausgeführt werden.

Lassen wir's mal mit den Beispielen...
Das ganze Ding sieht so aus, daß ich ein Hauptprogramm habe, was in einer Schleife einzelne Scripte nacheinander einbindet, in denen diverse Variablen gesetzt werden.

So wird z.B. in einem Script eine Variable $maketarget auf "&& make arm" gesetzt. Im Hauptprogramm wird dann 'make $maketarget' aufgerufen. Es soll also 'make && make arm' ausgeführt werden.

Wenn man das mit Backticks macht, würde erst '&& make arm' und dann 'make' ausgeführt, und das klappt nicht.

In den meisten Fällen, wo ein simples make reicht, wird diese Variable einfach leer sein. Deshalb wollte ich diese Ausnahme ohne großen Aufwand irgendwie unterkriegen.
Der Umweg über eine temporäre Datei funktioniert ja jetzt, aber ich hatte die Hoffnung, daß die Bash da doch noch ein paar mir unbekannte Möglichkeiten bieten könnte, um das eleganter zu lösen.
