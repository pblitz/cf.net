Wieder was für mein älteres Notebook mit Arch Linux - Scannen auf Befehl - und der Befehl wird direkt am Scanner ausgelöst ...

... Voraussetzung ist natürlich das man auf dem System überhaupt Pakete selber bauen kann respektive kompilieren kann - wie das geht steht hier *3

... gelöst wird das Ganze mit scanbuttond - leider seit scheinbar seit einiger Zeit nicht mehr weiter entwickelt - aber mein Scanner taucht noch unter "Supported Scanners" auf ... und das sollte natürlich bei jedem, der es probiert, der Fall sein ...


zunächst ein kleines Problem - mit der unter Arch installierten Version von libusb-compat klappt es wohl nicht - aber es gibt Abhilfe ...

... mit folgender PKGBUILD Datei (gefunden hier *2) ...

Code:
# $Id: PKGBUILD 101197 2010-11-28 15:10:38Z tpowa $
# Maintainer: Tobias Powalowski <tpowa@archlinux.org>
# Contributor: arjan <arjan@archlinux.org>

pkgname=libusb-compat
srcname=libusb
pkgver=0.1.12
pkgrel=1
pkgdesc="Library to enable user space application programs to communicate with USB devices"
arch=('i686' 'x86_64')
depends=('sh')
url="http://libusb.sourceforge.net/"
license=('LGPL')
source=(ftp://ftp.slackware.at/slackware-11.0/source/l/libusb/libusb-0.1.12.tar.gz)
options=('!libtool')

md5sums=('caf182cbc7565dac0fd72155919672e6')

build() {
  cd ${srcdir}/${srcname}-${pkgver}
  ./configure --prefix=/usr
  make
}

package() {
  cd ${srcdir}/${srcname}-${pkgver}
  make DESTDIR=${pkgdir} install
}



diese speichert man sich in einem temporären Verzeichnis als "PKGBUILD" und führt nacheinander folgende Befehle aus:

Code:

makepkg
makepkg -i


anschließend installiert man sich via yaourt das Paket scanbuttond aus dem AUR

Code:

yaourt -S aur/scanbuttond



jetzt sollte das Wiki von Gentoo weiterhelfen ... *4


noch ein Tipp am Rande - wenn der Scanner (wie bei mir auch) nicht immer an ist, müllt scanbuttond das Syslog voll - Abhilfe schafft folgendes Script - gefunden hier *5

Code:

#!/bin/bash
# only start scanbuttond if scanner is on, to avoid messages

scan_on=
while sleep 20; do
scan=`lsusb|grep Perfection`
if test -n "$scan"; then
if test -z "$scan_on"; then
/etc/rc.d/scanbuttond start
scan_on=true
fi
else
if test -n "$scan_on"; then
/etc/rc.d/scanbuttond stop
scan_on=
fi
fi
done </dev/null >/dev/null 2>&1 &
