Moinsen,

habe eine Idiotensichere Internetstation auf einem Rechner installiert (Debian Sarge, nur die Buttons: "Internet Verbinden, Email, Browser, Ausschalten)

Druckerinstallation hat noch nicht hingehauen, weil apt-get wieder unsinnig gnome-teile deinstallieren will damit ich cupssys instalieren kann.

Wollte daher zumindest die Möglichkeit eröffnen den Ausdruck auf einem anderen PC zu erledigen.

Problem: Es muss ausreichen, dass der Anwender den Knopf "Drucken" im jeweiligen Programm drückt.

Alles was ich bisher gefunden habe, gefiel mri wg. der einfachen Realisierung sehr gut (keine Sambainstallation etc.) erforderte jedoch immer mehrere Schritte:

Zitat:

ich mache meistens ein ps file am betreffenden rechner (mit

user@mailhost$ a2ps [otheroptions] -o psfile

) und drucke dann dieses file aus (eg. mit

user@localmachine$ ssh user@mailhost cat psfile | lpr


oder

Zitat:

das printcommand modifizieren dass es auf STDOUT druckt und dann mit
netcat auf einen listener schieben:

a2ps - -o - | nc -l -p 12342

das solltest du dann local abholen koennen und ausdrucken:

lpr <(nc mailhost 12342)

macht kein temporaeres file, dafuer hast du temporaer deinen ausdruck
"public" gemacht und nochdazu unverschluesselt uebers netz geschoben.

die einfachste und zu empfehlenste variante funktioniert wenn du vom
mailhost einen backconnect via ssh machen kannst. dann kannst du als
printcommand folgendes benutzen:

a2ps -o - - | ssh user@localmachine lpr

Weil der Router sich auch noch als Gateway vorstellt und dir den DNS mitteilt - die Infos musst du dann logischerweise auch von Hand einstellen.

ggf. so:
Code:
route add default gw <router-ip>


Der DNS-Server steht in /etc/resolv.conf

Warum hast du keinen Zugriff auf den Router? Du solltest schon eine IP fest vergeben, die außerhalb des Bereichs liegt, in dem der Router dynamisch IPs verteilt.
