Grundaetzlich kann ich es natuerlich nicht gutheissen, dass jemand VB lernen will

Grundsaetzlich ist aber zu libraries zu sagen, dass sie nichts weiter sind als eine Sammlung von bereits compilierten Funktionen.
Es macht also keinen Unterschied, ob du deine Funktion in den Quelltext schreibst oder in dieser library hast.
Deswegen: Wenn deine Funktion einen String liest und ein array zurueckgibt und das ganze auch noch das macht was es soll, dann funktioniert das auch in der library.

Es geht mir auch nicht nur darum ob Regex.Match nun True oder False ist: ich brauche die Werte der einfangenden Klammern.
Hier der Ultrageheime Reguläre Ausdruck:

Code:
 ^(?:([A-Z]{1,2})[-\s]+)(\d{4,5})(?:([^0-9].+)|$)

Code:
"Treffer gefunden"

Code:
"CH", "1234" und " Weltmeisterstadion"

CLI heisst nichts weiter als "Command Line Interface", also einfach ein Programm wie ipconfig (ein anderes Windows CLI Programm faellt mir gerade nicht ein)

das nimmt dann deinen String als Argument und spuckt dein array auf den stdout. Dazu muss es in Cobol nur eine system() Funktion geben (kann auch anders heissen) mit der sich andere Prozesse starten lassen.

Guenstigste Sprache fuer eine DLL: C, dafuer gibts auch eine Regexp library: http://gnuwin32.sourceforge.net/packages/regex.htm

Allerdings nur fuer bissel Regular Expressions halte ich das fuer overkill. Kenn mich mit Cobol nicht aus, aber gibts dafuer kein Regexp package/include/library/whatever?
