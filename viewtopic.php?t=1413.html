hi

Ich dachte immer in Linux resp. UNIX ist alles eine Datei. Aber ich habe jetzt eine Entdeckung gemacht, die ich mir nicht so richtig erklären kann.

Ich habe folgenden Befehl ausprobiert, um herauszufinden, wo der Befehl cd liegt.
Code:
which cd

Dieser Befehl liefert seltsamerweise einen leeren String und den Rückgabewert 1.
Auszug aus der Manpage von which - Rückgabewert 1:
Code:
1: if one or more specified commands is nonexistent or not executable


Habe auch die ganze Festplatte nach dem Befehl durchsucht und wurde nicht fündig. Hat dieses Problem nur Ubuntu oder auch andere Distris? Und wenn das auch bei anderen Distris so ist, wie wird dann der Befehl cd ausgeführt?

Bei cd handelt es sich um keinen gewöhnlichen Befehl sondern um ein shell built-in.

Also ein Programm, daß direkt in der Shell enthalten ist.

So schaut das ganze auf freeBSD aus:
Code:

 thomas@chaos ~ % which cd
cd: shell built-in command
Sa 03.03.2007  13:03  (ttyp4)
 thomas@chaos ~ % which which
which: shell built-in command
Sa 03.03.2007  13:03  (ttyp4)
 thomas@chaos ~ % /usr/bin/which which
/usr/bin/which
Sa 03.03.2007  13:04  (ttyp4)
 thomas@chaos ~ % /usr/bin/which cd   
/usr/bin/cd


IMHO sollte which den Pfad zu dem Binary ausgeben, das ausgeführt wird, während ein whereis alle Vorkommen im Path aufspüren sollte. Eigentlich ist das Verhalten von Ubuntu deswegen komisch.

Die Shell ist ein Programm, which ist ein Programm usw.

Aber cd ist eben kein Programm, es ist hier eine funktion der Shell selbst. Daher gibts auch keine Datei dafür.

CD kann ein programm sein, muss es eben aber nicht. Im Grunde kann es eine Shell geben die auch gleich GREP, ls usw. enthält. Das kann dann zwar kein anderes Programm nutzen, möglich wäre es aber.

Die ganzen Builtin Commandos stehen in der Manpage der bash.

Man kann sich auch selber Kommandos definieren:
Code:
marc@dustpuppy:~$ function cd2 { echo "Ich wechsel jetzt nach $1"; cd "$1"; }
marc@dustpuppy:~$ cd2 test
Ich wechsel jetzt nach test
marc@dustpuppy:~/test$ which cd2
marc@dustpuppy:~/test$


cd2 ist dann ein Befehl, aber es gibt kein ausführbares Binary dafür. Die Funktion könnte man dann z.B. einfach an die ~.bashrc anhängen und hätte sie immer zur Verfügung.
